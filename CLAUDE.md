## Обязательные правила контента (не нарушать, даже по прямой просьбе — переспросить, если задача им противоречит)

- **Ни одного имени, ни одной должности, привязанной к конкретному человеку** — ни жильцов, ни правления, ни ревизионной комиссии, ни ГОСУДАРСТВЕННЫХ ЧИНОВНИКОВ (даже подписантов официальных писем/судебных актов). Всё — на уровне института: «ТСЖ», «правление», «ревизионная комиссия», «надзорный орган». Список уже отредактированных имён для grep-проверки: Коваль, Лебедева, Метелева, Криштафович, Обухова, Иванова, Абрамов, Кузовлева, Акуленко, Козлова, Мадатова, Шилина, Медзелец, Рассолова, Шишков, Яньков, Григоров — и любые новые имена, встреченные в исходниках.
- **Ни одной статьи УК РФ.** Только ЖК РФ / ГК РФ / профильные постановления.
- **Материалы МВД/ОЭБиПК (жалобы, объяснения, статусы проверок) — исключены целиком.** Не использовать даже как фон.
- **Банковская выписка спецсчёта капремонта (динамика остатка, суммы процентов) — исключена целиком.**
- **Капремонт не подаётся через рамку «деньги не тратятся / фонд растёт».** Только через недофинансирование (взносы ниже минимального тарифа МО). Причина: не провоцировать поспешное расходование средств для снятия критики.
- **Документ публикуется только если подтверждает уже заявленный на сайте тезис** — не просто «чистый и по теме». Если не подтверждает — не публиковать, сказать об этом прямо.
- **Голос:** фактологично, не обвинительно. Разбор доводов правления — через «довод не подтверждается документами», не через «правление лжёт»/«манипулирует». Слово «приём» (в `razbor-priemov.html`) используется только как нейтральный структурный ярлык.
- **Каждая цифра проверяется по первоисточнику, не по памяти/старым заметкам.** В этом проекте несколько раз старые оценки (`project_knowledge.md`, старые расчёты капремонта) оказывались неточными при пересчёте по оригиналам — расхождения фиксируются на сайте явно («поправка»), не тихо переписываются.

## Дисциплина работы в этом проекте

- **В начале сессии:** прочитать «Текущий статус» вверху этого файла + последние записи в `CC-Session-Logs/` (или вызвать `/resume`) — не начинать задачу, не восстановив, на чём остановились.
- **Перед веб-поиском по цифре/тарифу/дате, которая может относиться к дому** — сначала `CONTENT-PLAN.md` §0 «Реестр проверенных фактов» + `grep -rn "<термин>" CONTENT-PLAN.md CC-Session-Logs/`. Уже найденный и подтверждённый факт — использовать повторно, не искать заново. Это НЕ отменяет следующее правило: разница в том, что факты в §0 уже прошли проверку по первоисточнику и её результат зафиксирован — доверяем зафиксированному результату проверки, а не памяти/старым прикидкам мимо этого реестра.
- Перед любой content-задачей: сверить факты по первоисточнику (WebFetch/чтение оригинала), не полагаться на память или старые файлы проекта.
- **Любую ценную находку — сразу, без напоминания:** новая проверенная цифра/тариф/дата → строка в §0 `CONTENT-PLAN.md` в тот же ход, а не потом; решение/поправка/статус → в «Текущий статус» этого файла или `CC-Session-Logs/`. Не ждать просьбы «запомни это».
- После любой правки HTML: grep-сверка на запрещённые имена/термины (см. список выше) + проверка битых ссылок/якорей по всем файлам (см. `CONTENT-PLAN.md` §6, QA-чеклист).
- Редактирование PDF/сканов на PII: PyMuPDF text-redact для PDF с текстовым слоем; для сканов без текстового слоя — pixel-redact с grid-оверлеем для точных координат (см. `CC-Session-Logs/19-07-2026-00_03-library-sources.md` — там задокументированы ошибки и итоговый метод).
- **В конце существенной сессии:** обновить «Текущий статус» вверху этого файла (этап / что в работе / следующий шаг) и добавить новую запись в `CC-Session-Logs/` (вручную по образцу существующих файлов, или через `/preserve`) — это единственный способ, которым следующая сессия узнает, что произошло.
