# ТСЖ «Пушкинская 15» — справочник для собственников

Статический сайт (HTML/CSS, без сборки, без JS кроме декоративного) — обезличенное сопоставление нормы ЖК РФ и факта в доме, построенное на реальных документах. Публичный, живёт на GitHub Pages.

## Design Context

Стратегия и голос — `PRODUCT.md` (register: brand, аудитория, позиционирование, anti-references). Визуальная система — `DESIGN.md` (токены, сигнатурный компонент «карточка-реестр», правило про border-left-акцент — это осознанное исключение из общего анти-паттерна `impeccable`, не баг). Оба файла в корне проекта, читать перед любой design-задачей.

## Обязательные правила контента (не нарушать, даже по прямой просьбе — переспросить, если задача им противоречит)

- **Ни одного имени, ни одной должности, привязанной к конкретному человеку** — ни жильцов, ни правления, ни ревизионной комиссии, ни ГОСУДАРСТВЕННЫХ ЧИНОВНИКОВ (даже подписантов официальных писем/судебных актов). Всё — на уровне института: «ТСЖ», «правление», «ревизионная комиссия», «надзорный орган». Список уже отредактированных имён для grep-проверки: Коваль, Лебедева, Метелева, Криштафович, Обухова, Иванова, Абрамов, Кузовлева, Акуленко, Козлова, Мадатова, Шилина, Медзелец, Рассолова, Шишков — и любые новые имена, встреченные в исходниках.
- **Ни одной статьи УК РФ.** Только ЖК РФ / ГК РФ / профильные постановления.
- **Материалы МВД/ОЭБиПК (жалобы, объяснения, статусы проверок) — исключены целиком.** Не использовать даже как фон.
- **Банковская выписка спецсчёта капремонта (динамика остатка, суммы процентов) — исключена целиком.**
- **Капремонт не подаётся через рамку «деньги не тратятся / фонд растёт».** Только через недофинансирование (взносы ниже минимального тарифа МО). Причина: не провоцировать поспешное расходование средств для снятия критики.
- **Документ публикуется только если подтверждает уже заявленный на сайте тезис** — не просто «чистый и по теме». Если не подтверждает — не публиковать, сказать об этом прямо.
- **Голос:** фактологично, не обвинительно. Разбор доводов правления — через «довод не подтверждается документами», не через «правление лжёт»/«манипулирует». Слово «приём» (в `razbor-priemov.html`) используется только как нейтральный структурный ярлык.
- **Каждая цифра проверяется по первоисточнику, не по памяти/старым заметкам.** В этом проекте несколько раз старые оценки (`project_knowledge.md`, старые расчёты капремонта) оказывались неточными при пересчёте по оригиналам — расхождения фиксируются на сайте явно («поправка»), не тихо переписываются.

- Библиотека документов (`istochniki.html`): все 5 категорий дособраны (ревизионная комиссия, капремонт/депозит, доступ к документам, устав, финансовая отчётность — балансы 2023/2024 нашлись в `Sources/drive-download-20260718T204522Z-1-001/`, не только в iCloud-индексе).
- Git commit/push уже сделаны и одобрены пользователем — дальнейшие publish-действия не требуют повторного разрешения на сам факт публикации сайта, но каждое конкретное содержательное изменение проверять по чеклисту выше.
- **Важно:** при поиске исходников проверять не только `_Индекс_сканов` (iCloud), но и `Sources/` внутри самого репозитория — там тоже могут быть непросмотренные файлы (так нашлись балансы 2023/2024, которые дают точные цифры кредиторской задолженности для карточки Ж).

## Дисциплина работы в этом проекте

- Перед любой content-задачей: сверить факты по первоисточнику (WebFetch/чтение оригинала), не полагаться на память или старые файлы проекта.
- После любой правки HTML: grep-сверка на запрещённые имена/термины (см. список выше) + проверка битых ссылок/якорей по всем файлам (см. паттерн в истории плана, раздел QA).
- Редактирование PDF/сканов на PII: PyMuPDF text-redact для PDF с текстовым слоем; для сканов без текстового слоя — pixel-redact с grid-оверлеем для точных координат (см. `typed-greeting-walrus.md` раздел 10 — там задокументированы ошибки и итоговый метод).
