# Простой слой объяснений — план реализации

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Добавить разговорный "на пальцах" слой поверх строгого реестра сайта: короткие plain-summary врезки на 9 карточках `chto-proishodit.html` + отдельная страница `prostymi-slovami.html` с 4 сгруппированными историями, каждая с блоком "почему это опасно".

**Architecture:** Статический HTML/CSS сайт без сборки и без тестового фреймворка (см. `CLAUDE.md`). "Тесты" в этом плане = QA-метод проекта: `grep` на запрещённые имена/статьи УК РФ (список в `CLAUDE.md`) + проверка целостности якорей/ссылок + визуальная проверка структуры. Никакого нового кода/логики не пишется — только HTML/CSS-контент.

**Tech Stack:** HTML, CSS (переменные из `css/styles.css`), без JS, без сборки.

## Global Constraints

- Ни одного имени/должности конкретного человека — ни жильцов, ни правления, ни ревизионной комиссии, ни госслужащих. Список для grep-проверки: Коваль, Лебедева, Метелева, Криштафович, Обухова, Иванова, Абрамов, Кузовлева, Акуленко, Козлова, Мадатова, Шилина, Медзелец, Рассолова, Шишков, Яньков, Григоров.
- Ни одной статьи УК РФ.
- Капремонт — только через рамку недофинансирования, никогда через "деньги не тратятся/фонд растёт".
- Голос — фактологично, не обвинительно, даже в разговорном регистре: "довод не подтверждается документами", не "ТСЖ обманывает".
- Никаких новых фактов/источников — только пересказ уже проверенных на сайте цифр (см. `CONTENT-PLAN.md` §0 и существующие карточки `chto-proishodit.html`).
- Существующие `registry-card` (норма/факт/статус/документ) не переписываются — только дополняются новым абзацем сверху.
- Новый компонент `.plain-summary` не использует `border-left`-акцент (это исключение зарезервировано только за `registry-card`/`callout`/`verdict-box` семьёй по DESIGN.md — новый произвольный элемент под него не подпадает).
